# LocalTrader

A local, autonomous crypto trading bot for Binance. Rules-based by default. Claude AI layer activates automatically when you add a key — and upgrades itself (Haiku → Sonnet → Opus) as the bot earns income.

Inspired by Conway Research's [Automaton](https://github.com/Conway-Research/automaton) web4 survival architecture.

---

## Quick Start

### 1. Prerequisites

```bash
node --version   # Need v20+
npm --version    # Need v9+
```

### 2. Install

```bash
cd "D:\AI\AI Trading"
npm install
```

### 3. Configure

```bash
copy config\.env.example .env
```

Edit `.env`:
```env
BINANCE_API_KEY=your_key_here
BINANCE_SECRET=your_secret_here

# Optional — bot runs without this, add later when profitable
CLAUDE_API_KEY=

# Auto-generated by setup script
DASHBOARD_TOKEN=
```

**Binance API key setup (critical):**
- Go to Binance → Profile → API Management
- Enable: **Spot & Margin Trading** only
- **DISABLE**: Enable Withdrawals, Enable Universal Transfer
- Restrict access to your IP address

Edit `config/genesis.json` to set your symbols, initial balance, and strategies.

### 4. Setup

```bash
npm run setup
```

This will:
- Validate your config
- Initialize the database
- Test your Binance connection
- Verify API key has NO withdrawal permission
- Generate a `DASHBOARD_TOKEN`

### 5. Backtest first (recommended)

```bash
# Seed historical data
npm run backtest -- --symbol BTC/USDT --strategy momentum --days 90 --seed

# Run backtest
npm run backtest -- --symbol BTC/USDT --strategy momentum --days 90
```

### 6. Paper trade (run for at least 30 days before going live)

In `config/genesis.json`, keep `"paper_mode": true`, then:

```bash
npm run dev
```

Dashboard: **http://127.0.0.1:3000**

### 7. Go live

When you're satisfied with paper trading results:
1. Set `"paper_mode": false` in `genesis.json`
2. Start with 10% of intended capital
3. Run: `npm start` (uses compiled build)

## Heartbeat Loops

| Loop | Interval | Purpose |
|------|----------|---------|
| positionMonitor | 30s | Stop-loss, take-profit, trailing stop ratchet |
| marketScanner | 60s | Fetch candles, compute indicators, generate signals |
| economicCheck | 300s | Update balance, tier transitions, purge old signals |
| strategyRotation | 900s | Detect market regime, switch strategy if needed |

## Survival Tiers

| Tier | Balance | Behaviour |
|------|---------|-----------|
| Normal | >70% initial | All strategies, full position size |
| Cautious | 50–70% | DCA only, 70% size, cancel pending entries |
| Critical | 25–50% | No new entries, manage existing only |
| Stopped | <25% | Emergency liquidation → halt |

---

## Circuit Breakers

1. **Daily loss >5%** — 24h pause
2. **Monthly drawdown >15%** — Liquidate 50% of positions (worst first), pause
3. **API error rate >5%** — 10 min pause
4. **Exchange disconnect >2min** — Cancel pending entries (OCO stops stay on exchange)
5. **STOP file** — Immediate emergency liquidation of all positions

**Emergency stop:** Create a file named `STOP` in the project root. The bot halts within 30 seconds.

## AI Integration

Works without a Claude API key. When you add one, the model auto-selects by account balance:

| Balance | Model | Use |
|---------|-------|-----|
| At initial | Haiku | Signal validation (cheapest) |
| +10% above initial | Sonnet | Strategy confirmation |
| +50% above initial | Opus | Deep analysis |

**Daily spend cap** always overrides balance-based upgrades — if cap is hit, drops to Haiku for the rest of the day.

Set `CLAUDE_API_KEY` in `.env` to enable. Set `ai.max_daily_spend_usd` in `genesis.json` to control costs.

Available at **http://127.0.0.1:3000** (localhost only — not accessible from other devices).

| Page | URL | Content |
|------|-----|---------|
| Overview | `/` | Balance, PnL, open positions, tier badge |

| Risk | `/risk` | Drawdown chart, circuit breaker status |

## Philosophy

From `config/soul.md`:

> Preserve capital above all else. Profitability is secondary to survival.

This bot is not a get-rich-quick machine. It is a capital preservation and systematic growth system. Every circuit breaker, every survival tier, every stop-loss exists to protect your capital during adverse conditions.

Run paper mode seriously. Treat simulated losses as real. Only go live when the numbers are consistent.
